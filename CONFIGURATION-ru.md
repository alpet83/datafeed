## НАСТРОЙКА ЗАГРУЗЧИКОВ ДАННЫХ

Пароли и хосты БД должны быть прописаны в lib/db_config.php для соответствующих пользователей. Пользователей можно создать используя шаблоны sql/create_user*.sql

Все настройки данных для скриптов огранизуются в таблицах ticker_map и data_config. По умолчанию такие таблицы создаются вместе с БД, при использовании прототипов sql/db_****_proto.sql. 
В первой таблице указывается соответствие тикера (уникального для всех БД) и символа торговой пары, вместе с уникальным опять-же для всех БД pair_id. При этом id_ticker должен быть уникальным лишь в пределах данной таблицы, он используется для связывания с data_config. Конфигурация загрузки предполагает сочетание двух флагов: 1 - загрузка истории, 2 - загрузка последних данных (для чего может подключаться WebSocket API). 


Для общей статистики и данных CoinMarketCap требуется также БД datafeed, которая может быть создана через шаблон cm_tables.sql. Чтобы сохранять данные CoinMarketCap потребуется прописать API ключ в /etc/CMC-API.key

### Актуальные замечания по runtime

- Основной рабочий режим сейчас ориентирован на PHP 8.4 CLI.
- Tick loaders используют ClickHouse как основное хранилище больших массивов тиков. Если `CLICKHOUSE_HOST` не задан, tick loader может завершиться предупреждением `skipping ticks loader run` без фатальной ошибки.
- Candle loaders хранят минутные свечи в MariaDB и дополнительно используют дневные таблицы вида `candles__<ticker>__1D` для проверки объема и коррекции блоков.
- Перед включением candle loader в cron нужно проверить, что таблицы `candles__<ticker>__1D` существуют и доступны на запись. При их отсутствии загрузчик может входить в цикл повторных неуспешных импортов.

### Ограничение истории через переменные окружения

Поддерживаются глобальные и exchange-specific ограничения:

- `DATAFEED_HISTORY_LIMIT_DAYS`
- `DATAFEED_HISTORY_MIN_TS`
- `DATAFEED_HISTORY_LIMIT_DAYS_<EXCHANGE>`
- `DATAFEED_HISTORY_MIN_TS_<EXCHANGE>`

Примеры суффиксов: `BNC`, `BFX`, `BMX`, `BBT`.

Эти переменные используются загрузчиками для смещения нижней границы исторической подкачки без изменения исходного кода.
